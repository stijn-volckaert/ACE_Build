Setting up a new version:
1) Edit GameServer/Classes/ACEInfo.uc to exclude previous version
2) In PlayerManager build VM, run setupBuildFiles.rb, setupNative.rb, setupUScript.rb
3) In VS 2017, edit Debug project settings for Client, GameServer, PlayerManager projects. Update dll names in General and Linker>General

Full ACE build:
1) Build GameServer Release build in VS
2) Build PlayerManager Release build in VS
3) Build Client Release-NonSSE, Release-SSE2, Release-AVX2 builds in VS
4) In Linux GameServer VM, navigate to UTWindows/ACE/GameServer/Src, then run make
5) In Linux PlayerManager VM, navigate to UTWindows/ACE/PlayerManager
6) If build folder already exists, cd build and skip to step 9
7) mkdir build && cd build
8) cmake ../ -DCMAKE_BUILD_TYPE=Release -DCMAKE_CXX_FLAGS="-O3 -fPIE -pie -std=c++11"
9) make -j 8
10) In Windows, navigate to UT/System in terminal, then run compace.bat

Server-only build:
1) Build GameServer Release build in VS
2) Build PlayerManager Release build in VS
3) In Linux GameServer VM, navigate to UTWindows/ACE/GameServer/Src, then run make
4) In Linux PlayerManager VM, navigate to UTWindows/ACE/PlayerManager
5) If build folder already exists, cd build and skip to step 8
6) mkdir build && cd build
7) cmake ../ -DCMAKE_BUILD_TYPE=Release -DCMAKE_CXX_FLAGS="-O3 -fPIE -pie -std=c++11"
8) make -j 8
9) In Windows, navigate to UT/System in terminal, then run compaceserver.bat

Debugging ACE on Windows:
1) Build GameServer Debug build in VS
2) Build PlayerManager Debug build in VS
3) Build Client Debug build in VS
4) Navigate to UT/System in terminal, then run compacedebug.bat
